# Users & Workflows

## User Personas

### CSE (Client Success Engineer)
- **Goal**: Manage 10-20 client relationships daily
- **Frequency**: Multiple times per day
- **Key tasks**: Check client health, log meeting notes, track actions, review segmentation compliance
- **Pain points**: Manually cross-referencing Excel files, forgetting follow-ups, no single source of truth
- **Success metric**: Every client interaction is tracked, nothing falls through the cracks

### CAM (Client Account Manager)
- **Goal**: Drive revenue retention and growth across assigned accounts
- **Frequency**: Daily
- **Key tasks**: Pipeline tracking, renewal forecasting, identify upsell opportunities, prepare for exec meetings
- **Pain points**: Scattered data across BURC Excel and CRM, manual revenue calculations
- **Success metric**: Accurate pipeline visibility, proactive risk identification

### APAC Leadership
- **Goal**: Portfolio-level visibility of business vs targets
- **Frequency**: Weekly/fortnightly
- **Key tasks**: Review ARR vs target, identify at-risk clients, track team performance, exec briefings
- **Pain points**: Asking CSEs for updates, waiting for finance to update spreadsheets
- **Success metric**: Self-service dashboard with real-time portfolio health

## Core Workflows

### CSE Morning Routine
```
1. Open dashboard → see "My Portfolio" health summary
2. Check overdue actions → prioritise follow-ups
3. Review upcoming meetings → prep with client context
4. Check ChaSen AI suggestions → act on proactive nudges
5. Review segmentation compliance → ensure operating rhythm on track
```

### Client Deep Dive
```
1. Navigate to client detail page
2. See health score, NPS, ARR, contract status at a glance
3. Tab: Meetings → recent notes, upcoming scheduled
4. Tab: Actions → open items, overdue, completed
5. Tab: Financials → revenue breakdown from BURC
6. Tab: News → relevant industry articles
7. Tab: Sentiment → communication tone trends
```

### Meeting Workflow
```
1. Before: Review client context, ChaSen suggests talking points
2. During: (Future) Live transcription + co-host suggestions
3. After: Log meeting notes → AI extracts actions → actions auto-created
4. Follow-up: Actions tracked, reminders sent, ChaSen follows up
```

### Exec Briefing
```
1. Auto-generated weekly executive briefing
2. 12 data sections: portfolio health, financials, NPS, support, pipeline
3. AI summary with coaching tone
4. Optional audio briefing (OpenAI TTS)
5. Available on-demand via dashboard or daily digest email
```

## What "Done" Looks Like

The platform succeeds when:
- A CSE never needs to open the BURC Excel file directly
- Leadership can answer "how are we tracking?" without asking anyone
- Meeting follow-ups happen automatically, not through memory
- At-risk clients are flagged before they escalate
- The data on screen matches the source spreadsheets exactly
